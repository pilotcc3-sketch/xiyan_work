D1 ★ 模板池信息架构演化：两 Tab → 三 Tab → 创意广场独立 + 模板池保留双 Tab
已锁定
📅 2026-05-29 👤 产品·小樊 🔗 影响：模板池主信息架构
背景
现有模板管理平台是「草稿」+「正式」两 Tab。基座希望"AI in ALL · 鼓励全员体验生成制作模板"，但完全开放写入正式管理体系会破坏管理秩序——草稿一旦写入即占用资源、影响搜索、可被其他模板复用。
候选选项
  • A · 不变 · 仍然两 Tab，全民生成的模板直接进草稿池（管理混乱风险高）
  • B · 增设独立"全民生成"入口/页面，与平台割裂（割裂感强）
  • C · ✓ 模板池演化为三 Tab：💡创意广场 + 📝草稿 + ✅正式（创意广场前置，鼓励全民提案；草稿池保留产品负责制；正式池保持高标准）
决策内容
模板池主视图调整为三 Tab：
  • 💡 创意广场：所有角色都可以一句话/截图/Figma 提案，配场景/赛道筛选 + 点赞/评论；提案不直接入库。
  • 📝 草稿：分为"待入库审批"和"已入库未上线"两类；已入库的可被复用、被搜到。
  • ✅ 正式：已上线 / 已下线归档。

📌 后续演化（v0.2 路演态）：三 Tab 拆分为两个独立板块放在「首页」内——
  • 💡 创意广场独立成一个板块（鼓励全员提案的氛围更强）；
  • 📦 模板池独立成一个板块，内部仍保留「📝 草稿 / ✅ 正式」双 Tab（管理秩序更清）。
两个板块仅在「🏠 首页」Tab 可见；「👤 我的工作台」Tab 内不显示，避免认知噪音。
影响 / 后续
· 配套 D2 双审批：创意广场提交后必须经"草稿入库审批"才能进草稿池。
· 创意广场设计采用「中档」方案：灵感卡片墙 + 场景/赛道筛选 + 点赞/评论（不做 AI 推荐相似已上线模板）。
· 与基座侧栏"看板卡片"协同：创意广场的提案数会成为大盘看板的一项指标。
信息架构模板池全民共创差异化用户层
D3 ★ 设计-研发冲突解法 = 对齐器嵌入流水线 ②，不做"以谁为准"的强约束
已锁定
📅 2026-05-29 👤 产品·小樊 🔗 影响：流水线步骤②
背景
历史顽疾："设计稿和最终上线效果不一致"——设计师怪研发不还原、研发怪 DSL 能力有限、产品两头夹。常见解法是"以谁为准"的强约束（要么强制还原 Figma 牺牲性能，要么以 DSL 能力为上限牺牲设计），都不解决问题。
候选选项
  • A · 强约束 · 以 Figma 为准，不达成上线不允许（研发反弹大）
  • B · 强约束 · 以 DSL 为准，设计自适应（设计反弹大）
  • C · "轻档"对齐器 · 仅文字描述差异（说服力不够）
  • D · ✓ "中档"对齐器 · 在轻的基础上加 diff 视图（Figma 视觉 vs DSL 渲染并排，差异点高亮，逐项可选"采纳设计/保留 DSL"）
  • E · "重档"对齐器 · 自动 AI 修复（成本高、Demo 难讲）
决策内容
对齐器（Skill: 设计稿对齐器 · B 级）嵌入到流水线的第②步「设计对齐」，不做独立模块入口。形态为：左 Figma 视觉源、右 DSL 实时渲染、下方 diff 列表（差异 #1 主图圆角 / #2 副标题色…），每项支持"采纳 Figma / 保留 DSL"二选一。不强制对齐才能进下一步，但所有未对齐项会被记录在"上线审批"的设计 Review 视图里。
影响 / 后续
· 后续可扩展到「截图对齐 / 视频对齐 / 竞品对齐」（迭代规划）。
· 与上线审批联动：设计 Review 时直接看 diff 列表，省去"逐像素对比"的耗时。
· 是本模块"AI 原生"差异化的最强卖点之一。
流水线对齐器协作模式差异化合作层
D5 ★ 不重做 iDOMO 画布，本模块定位为"上层壳"
已锁定
📅 2026-05-29 👤 产品·小樊 🔗 影响：模块边界 / 兼容性
背景
团队工作原则之一是"零基思维 + AI 重构基础设施"，但完全推翻 iDOMO 画布意味着：① 画布引擎能力（动效、分发、SDK 兼容）需重写至少 3-6 个月；② 历史模板迁移成本不可控；③ 抢占了引擎团队的领域。和 MVP 6/30 严重冲突。
候选选项
  • A · 完全重做 · 抛弃 iDOMO 画布，从零写新引擎（不可行）
  • B · 渐进替换 · 部分场景试点新画布（边界模糊、易扯皮）
  • C · ✓ iDOMO 引擎是底座，本平台是上层壳（边界清晰、风险可控、释放精力做协同）
决策内容
本模块明确定位为「样式生产工作壳」——iDOMO 引擎做底层画布，本平台在其上叠加：
  • 双入口 Agent（标准流 / 零稿流）
  • Skills 拓展（D2C / C2D / 体检 / 对齐 / 测试 / 成绩单）
  • 对齐器（Figma vs DSL diff）
  • 双审批（草稿入库 + 上线发布）
  • 协同视图（首页 + 我的工作台 + 创意广场）
  • 自动成绩单（卡片报告 + 模拟企微推送）
Demo 中所有"画布编辑"的场景都明确标注"由 iDOMO 引擎承载"。
影响 / 后续
· 兼容性最强：存量模板"读但不迁"，可一键跳回 iDOMO；新模板默认走新流。
· 与引擎团队的协作清晰：本模块通过 iDOMO API 操作 DSL 模板，不染指引擎本身。
· MVP 时间风险大幅降低。
模块边界兼容性底座关系兼容性差异化
D2 审批从"单道"升级为"双道"——草稿入库 + 上线发布
已锁定
📅 2026-05-29 👤 产品·小樊 🔗 配套 D1
决策内容
在 iDOMO 现有"上线审批"之上，新增"草稿入库审批"。任何角色（包括运营/设计/研发）通过 Agent 或创意广场生成的模板，会被打上「待入库」徽章，必须经产品角色审批通过才能进入正式草稿池。

未入库前能做什么：
  • ✅ 编辑 / 预览 / 自动化测试
  • ❌ 不可进入公共模板库被搜到
  • ❌ 不可被其他模板复用为基础
  • ❌ 不可上线
审批流权限合作层

D8 首页 / 我的工作台默认首页
已锁定
📅 2026-05-29 👤 产品·小樊
决策内容
采用同页 Tab 切换，默认首页。理由：① 路演首屏冲击力首页（全员视角）> 我的（个人视角）；② 新用户首次进入对"我"无概念，先看首页建立全局；③ 老用户切换成本低（一个 Tab 点击）。
视角切换为运营时，默认改为聚焦创意广场。
