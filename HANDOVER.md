## 3. Demo 里的 8 步流水线（一定要理解）

```
① 模板创建 → ② 模板编辑 → ③ 模板测试 → ④ 草稿发布
    ↓
⑤ 规则配置 → ⑥ 模板实验 → ⑦ 全量上线 → ⑧ 数据成绩单
```

**关键概念**：
- **L1 / L2 双行**：主干（L1，已在线的稳定版本）+ 分支（L2，正在造的新一轮）。同一模板可以 L1 已经全量、L2 同时在造 v3。这是分支心智模型（详见 `clean/model.html`）
- **拦截逻辑**：每一步之间都有硬性拦截，如 ③→④ 必须点过"开始自动化测试"、②→③ 必须点过"打开 iDOMO 编辑器"。这些拦截都写在 `clean/index.html` 的 JS 里（搜 `pipeline-guard` 相关 commit）
- **⑥ 实验环节**：最新需求要把它从可选改成**必经**，配套一整套后台状态机 + 3 条审批流拆分（详见 `需求-实验环节重构.md`），这块是**下一位同学接手后大概率要推进的重点**
