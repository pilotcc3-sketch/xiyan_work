③ 已发布副本状态机

⚠️ 状态机只属于已发布副本。草稿副本只有"草稿"一种状态，无状态机。

入口：草稿页「草稿发布」MyOA 三段审批通过
↓
⑤
已发布-未生效
已进管理页，但还没配规则 / 还没投放
↓配规则 + 启动 AB 实验
⑥
已发布-实验中
AB 实验进行中，正在收集数据
↓实验结论 OK + MyOA 二段审批（产品/研发）
⑦
已发布-全量
全流量投放，看成绩单
提示："已发布"≠"在投放"。已发布只代表"它进了管理页、有线上身份"，是否真给量取决于⑤⑥⑦的子状态。仅已发布-全量对应可看成绩单。

④ 过桥事件细节 · 草稿发布 = 物理覆盖

覆盖之前

  • 📝 草稿副本：草稿版 · 编辑中
  • 🚀 已发布副本：线上版 · 6 月 5 日
  • 状态可能是 ⑤ / ⑥ / ⑦ 任意一个
⚡
审批通过

覆盖之后（瞬间发生）

  • 🚀 已发布副本：线上版 · 6 月 10 日（新）
  • 状态被强制重置为 ⑤ 已发布-未生效
  • 📝 草稿副本：仍在，可继续改下一版
四条硬规则：
  1. 把当前草稿 DSL 快照打包，物理覆盖已发布副本
  2. 旧"线上版 · 6/5" 不留底、不可回滚
  3. 如果旧线上版当时正在 ⑥ 实验中 → 强制中止旧实验（数据停止累积）
  4. 新"线上版 · 6/10"进入 ⑤ 未生效 → 必须重新配规则、重新跑实验

⑤ 时间并行性 · 双线同时演进

两份副本是并行存在、同时演进的两条线，不是串行的"先草稿后发布"。

6/1 6/3 6/5 ⚡发布 6/8 6/10 ⚡发布
📝 草稿副本
草稿版 · 6/1 创建 → 草稿版 · 6/3 编辑 → 6/5 发布 → 草稿版 · 6/7 改新版 → 6/10 发布 → 草稿版 · 编辑中…
🚀 已发布副本
（尚未进管理页） → 线上版 6/5 · ⑤未生效 → 线上版 6/5 · ⑥实验中 → 线上版 6/5 强制中止 → 线上版 6/10 · ⑤未生效
关键观察：当 6/10 草稿发布时，旧"线上版 6/5"还在 ⑥ 实验中 → 实验被强制中止 → 新"线上版 6/10"重新从 ⑤ 开始。实验数据需要重新跑。

⑥ 两条主流程

流程 A · 新模板首次上线（一次性）

① 创建 → ② 编辑 DSL → ③ 测试 → ⚡ ④ 草稿发布 草稿副本域
↓ 进管理页（首次）
⑤ 已发布-未生效 → ⑥ 已发布-实验中 → ⑦ 已发布-全量 已发布副本域

流程 B · 老模板迭代（高频循环 · 真实日常）

⑦ 旧线上版 6/5 在全量投放 已发布副本域
↓ 同时
📝 草稿副本继续改 → ② 编辑下一版 → ③ 测试 → ⚡ ④ 草稿发布 草稿副本域
↓ 物理覆盖（旧 6/5 死掉）
⑤ 新线上版 6/10 未生效 → ⑥ 重新跑实验 → ⑦ 新版 6/10 全量 已发布副本域
↻ 周而复始（每次草稿发布都覆盖一次）
