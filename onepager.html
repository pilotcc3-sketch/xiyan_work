一、产品层 5 题 · 路演必答

Q1 角色和定义我是谁，要交付什么
样式生产管理是从需求接入到上线复盘的端到端模板生产工作壳，承接 iDOMO Plus，但加厚了「AI 协同 / 双审批 / 对齐器 / 自动成绩单」这一层。
交付物：一个让 产品 / 设计 / 研发 / 运营 四角色都能上手的 AI 原生模板生产工作台。
Q2 产品设计骨架 / 北极星 / MVP
骨架四件套：
Agent 输入条 + 抽屉 首页 / 我的工作台 创意广场 + 模板池（草稿/正式） 8 步生产流水线
北极星指标：上线后 CTR 提升中位数。
MVP（6/30）：以「开屏」场景跑通"标准流"端到端：TAPD 链接 → 体检 → 建壳 → Figma D2C → 对齐 → 编辑 → 测试 → 双审批 → 上线 → 成绩单。
Q3 差异化和 iDOMO/样式工厂最大的不同

现状（iDOMO Plus / 样式工厂）

  • · 入口：功能堆叠，各角色各找各的
  • · 角色：研发主导，设计/运营被动
  • · 流转：线性提单 → 等待 → 提交 → 上线
  • · 反馈：上线后效果靠人去拉

样式生产管理（新）

  • · 入口：Agent 输入条 + 首页 / 我的工作台
  • · 角色：四角色同台 + 对齐器消解冲突
  • · 流转：双审批 + 创意广场前置 + 实验闭环
  • · 反馈：自动成绩单 + 模拟企微推送
Q4 兼容性怎么兼容历史 / 怎么过渡
· iDOMO 引擎不动，作为底层画布保留（决策 D5）。
· 存量模板"读但不迁"——保留只读视图，可一键跳回 iDOMO 编辑。
· 新模板默认走新流，老入口同步保留 90 天，灰度迁移。
· 规则配置、洞察归因、问题排查模块通过基座路由跳转，本模块不重做。
Q5 迭代性能力可拓展性 / 后续规划
· Skills 即拓展底座：人人可贡献，A/B/C 分级审核（A=平台审 / B=团队审 / C=自助）。
· 对齐器 从「Figma↔DSL」扩到「截图/视频/竞品」对齐。
· 成绩单 从「模板级」扩到「元素级」（与洞察归因模块联动归因到主图/标题/按钮）。

二、用户层 2 题

Q6 小白用户可用性是否易上手
一进首页只看到 Agent 大输入条 + 首页 / 我的工作台双 Tab，零学习成本。Agent 默认两 Tab 切换：
📋 标准流 · 粘 TAPD 链接 🌀 零稿流 · 粘构想/截图/Figma
每个状态都有"下一步该点什么"的明确引导。即便是非工具背景的运营、产品，也能在 5 分钟内提交第一个创意。
Q7 具体 case 解决典型用例如何走完
路演剧本（开屏场景）：
① 粘 TAPD 链接→ ② 自动体检→ ③ 一键建壳→ ④ Figma D2C→ ⑤ 对齐器 diff→ ⑥ 编辑预览→ ⑦ 自动化测试→ ⑧ 草稿入库审批→ ⑨ 上线审批→ ⑩ 灰度→ ⑪ 成绩单 + 企微推送

三、合作层 2 题

Q8 上下游谁给我输入 / 谁拿我输出
类型合作方接口形态
上游TAPD（需求）链接导入 + Skill 体检
上游Figma（设计）插件/链接导入 + D2C
上游竞品截图 / 灵感拖拽 + C2D
横向iDOMO 引擎底层画布（保留，不重做）
横向实验平台白名单 + 实验数据回流
横向企微成绩单卡片自动推送（机器人）
下游规则配置模块路由跳转 + 入参 templateId
下游洞察归因模块路由跳转 + ?action=optimize
下游问题排查模块异常徽章联动 + 一键跳转
Q9 配合模式四角色怎么协作
· 产品：TAPD 起单 → Agent 体检后一键拉入；本模块独有「待我审批」队列。
· 设计：Figma 上传进对齐器，参与上线审批，设计 Review 一票否决。
· 研发：在画布编辑、审计 Skills 安全级别、维护 DSL 能力。
· 运营：创意广场首要使用人，提案灵感 → 草稿入库审批进入正式流。
· 测试：自动化测试为主（多机型/多分辨率回归）+ 人工抽检高风险 Skill。

四、分模块小启发 · 生产 5 题 · 主考题

P1 模板需求如何输入
三入口合一在 Agent：① TAPD 链接（标准流 Tab）② 构想/截图（零稿流 Tab）③ 创意广场提案。 前两者直接进生产流水线，第三者经"草稿入库审批"后进入正式流。
P2 模板如何生产
D2C / C2D / Smart Prompt + Agent 微调，全程画布在 iDOMO 引擎。本平台是壳，做"准备物料 → 对齐 → 编辑 → 预览 → 测试 → 审批 → 上线 → 成绩单"八步流水线。 其中第②步嵌入对齐器（Figma vs DSL diff），第⑥步走双审批。
P3 模板状态如何流转
6 态明牌，每个状态都有显式徽章：
💡 创意→ 📝 草稿（待入库）→ ✏️ 正式草稿→ 🚦 待上线→ ✅ 已上线→ 📦 已下线/归档
P4 模板效果如何追溯
每个上线模板自动生成成绩单卡片：CTR / 消耗 / 实验提升 / 异常归因 / 调优建议 / 回滚按钮。 同时支持"模拟企微推送"视图（D9 决策），可一键发送到投放群。
P5 模板如何优化
成绩单中"调优建议"→ 一键克隆为 v2 草稿继承数据；可跳转洞察归因模块（带参数 ?templateId=&action=optimize），由洞察归因给出元素级归因建议后回流本模块继续生产。
